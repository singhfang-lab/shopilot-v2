# 验证标准规则

> 用途：所有建议都必须告诉商户“做完后怎么看有没有效果”。

## 1. 验证标准必须包含四要素

```text
观察周期 + 指标 + 有效标准 + 下一步判断
```

| 要素 | 示例 |
|---|---|
| 观察周期 | 7天、14天、30天 |
| 指标 | 订单数、客单价、套餐销量、毛利、差评、复购 |
| 有效标准 | 回升10%以上、没有新增差评、毛利下降不超过8% |
| 下一步判断 | 有效就放大，无效就回到下一类问题排查 |

---

## 2. 常见周期建议

| 动作类型 | 建议观察周期 |
|---|---|
| 门口展示/菜单前排调整 | 7-14天 |
| 套餐或加购测试 | 14天 |
| 差评整改 | 30天 |
| 老客复购动作 | 14-30天 |
| 社媒内容 | 30天 |
| 活动复盘 | 活动前7天 + 活动中 + 活动后7天 |
| 员工流程调整 | 7-14天 |
| 新品测试 | 14-30天 |

---

## 3. 指标选择规则

| 问题 | 主要验证指标 |
|---|---|
| 生意下降 | 订单数、客单价、销售额、差评 |
| 竞品应对 | 订单数、新客数、客单价、评分/评价 |
| 菜单优化 | 主推商品销量、客单价、点单速度、差评 |
| 套餐提高客单 | 套餐销量、客单价、毛利、加购率 |
| 复购 | 二次到店、老客占比、复购客单价 |
| 差评处理 | 同类差评数量、评分、新客咨询 |
| 活动效果 | 订单、毛利、新客、客单价、复购、差评 |
| 社媒内容 | 咨询量、到店提及、互动、预约/下单 |

---

## 4. 标准表达模板

### 模板一：7天短期测试

```text
先测试7天。7天后看：
- 订单数是否回升10%以上；
- 客单价是否没有明显下降；
- 相关差评是否没有新增。

如果订单回升但客单价下降，说明拉回来的主要是低价订单，需要补利润款或加购。
如果订单没有回升，再继续查竞品和门店展示。
```

### 模板二：14天套餐测试

```text
这个套餐先测试14天。14天后看：
- 套餐销量是否达到总订单的10%-15%；
- 客单价是否提升10%以上；
- 毛利是否没有明显下降。

如果套餐销量提升但毛利下降，说明套餐组合需要调整，不适合继续加大折扣。
```

### 模板三：30天口碑整改

```text
整改后观察30天。重点看：
- 同类差评是否减少；
- 评分是否稳定或回升；
- 新客咨询是否恢复；
- 好评中是否开始出现新的正向关键词。
```

---

## 5. 禁止无效验证表达

不要只写：

- 观察效果；
- 看数据变化；
- 关注订单；
- 后续复盘；
- 看有没有改善。

必须写清楚：

- 看哪个指标；
- 看多久；
- 什么结果算有效；
- 如果无效下一步怎么做。
