# 上线前检查清单

> 用途：在知识库进入 Gamma4 或正式商户助手前，做最终检查。

## 1. 文件结构检查

- [ ] 是否有全局回答协议；
- [ ] 是否有数据字典；
- [ ] 是否有商户数据输入模板；
- [ ] 是否有商户画像卡；
- [ ] 是否有问题意图识别规则；
- [ ] 是否有问题卡组合规则；
- [ ] 是否有公开数据使用规则；
- [ ] 是否有测试问题库；
- [ ] 是否有评分表；
- [ ] 是否有版本管理规则。

---

## 2. 问题卡检查

每张问题卡必须具备：

- [ ] 适用场景；
- [ ] 问题类型；
- [ ] 常见影响因素；
- [ ] 影响因素展开；
- [ ] 数据策略；
- [ ] 有数据时必须分析什么；
- [ ] 数据命中规则；
- [ ] 无数据版回答蓝图；
- [ ] 有数据版回答蓝图；
- [ ] To-do 颗粒度要求；
- [ ] 验证标准；
- [ ] 禁止表达；
- [ ] 合格/不合格示例。

---

## 3. 回答质量检查

测试至少覆盖：

- [ ] 无数据问题；
- [ ] 少数据问题；
- [ ] 汇总数据问题；
- [ ] 明细数据问题；
- [ ] 复合问题；
- [ ] 决策问题；
- [ ] 复盘问题；
- [ ] 行业适配问题。

每个回答必须：

- [ ] 先回答商户问题；
- [ ] 不暴露内部字段；
- [ ] 无数据时不强下结论；
- [ ] 有数据时给证据链；
- [ ] To-do 具体到对象、动作、周期、指标；
- [ ] 有验证标准；
- [ ] 避免未经验证假设；
- [ ] 避免空泛表达。

---

## 4. 典型风险检查

重点检查模型是否还会出现：

- [ ] 直接建议降价；
- [ ] 直接说做活动；
- [ ] 用“护城河、品牌格调、圈层裂变”等不落地词；
- [ ] 把公开数据当事实；
- [ ] 使用不适合海外场景的渠道词；
- [ ] 商户给了数据仍然只说泛建议；
- [ ] 没有验证标准；
- [ ] To-do 无法执行。

---

## 5. 上线门槛

建议上线前满足：

1. 20 张核心问题卡测试平均分 ≥ 28/35；
2. 有数据版回答必须明显优于无数据版；
3. 80% 以上回答具备具体 To-do；
4. 不合格回答已完成回炉记录；
5. 内容同事能按模板继续生产新问题卡；
6. 模型输出不再频繁出现空泛建议。

---

## 6. 上线后监控

上线后应持续记录：

- 商户真实问题；
- AI回答；
- 是否命中正确问题卡；
- 商户是否继续追问或补数据；
- 回答是否被人工判定为有用；
- 哪些问题卡命中率高；
- 哪些问题卡回答质量低。

每 2-4 周进行一次问题卡更新。
