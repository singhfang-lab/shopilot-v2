# 问题卡元数据规范

> 用途：让问题卡可管理、可检索、可测试、可迭代。

## 1. 每张问题卡建议包含 YAML 元数据

示例：

```yaml
id: diag_006
title: 多家门店中只有一家下降，为什么？
module: 经营诊断
question_type: 原因诊断型
industry_scope: 通用
data_dependency: 中
web_search_applicable: 部分适用
positioning_dependency: 弱
version: v1.0
status: testing
owner: 内容同事A
last_updated: 2026-04-26
related_cards:
  - diag_003_sales_decline_order_vs_aov
  - competitor_001_analyze_competitors
  - review_001_complaint_diagnosis
```

---

## 2. 字段说明

| 字段 | 说明 |
|---|---|
| id | 唯一编号，建议模块前缀 + 数字 |
| title | 问题卡标题 |
| module | 所属模块 |
| question_type | 原因诊断/判断决策/方案生成/方法教学/复盘评估/定位表达 |
| industry_scope | 通用或具体行业 |
| data_dependency | 低/中/高 |
| web_search_applicable | 不适用/部分适用/高度适用 |
| positioning_dependency | 弱/中/强 |
| version | 版本号 |
| status | draft/testing/approved/live/deprecated |
| owner | 内容负责人 |
| last_updated | 最近更新时间 |
| related_cards | 相关问题卡 |

---

| 模块 | 前缀 |
|---|---|

---

## 4. 状态流转

```text
draft → testing → approved → live → deprecated
```

| 状态 | 含义 |
|---|---|
| draft | 初稿，未测试 |
| testing | 正在用测试问题验证 |
| approved | 通过测试，可上线 |
| live | 已进入正式知识库 |
| deprecated | 已废弃或被新卡替代 |

---

## 5. 版本号规则

- v0.x：草稿；
- v1.0：首次测试通过；
- v1.1：小幅文案或示例修改；
- v2.0：结构、判断逻辑或输出契约重大修改。

---
