# 商户端回答蓝图库

本文件汇总六类商户端回答蓝图。每张问题卡会按其问题类型引用其中一种蓝图，并结合该问题的影响因素、数据命中规则和案例生成最终商户回答。

## 原因诊断型

### 无数据版回答结构
1. 先给初步判断，但不要装作已经确定原因。
2. 列出这类问题最常见的 3-5 个影响因素。
3. 解释这些因素在门店里通常怎么发生、会造成什么影响。
4. 把问题分成外部、内部执行、商品结构、体验口碑等可排查类别。
5. 告诉商户现在先查什么，并自然引导补充关键数据。
6. 给排查型 To-do：今天查什么、3 天内记录什么、7 天内验证什么。

### 有数据版回答结构
1. 直接给数据结论，说明核心问题是什么。
2. 用 2-5 条关键数据做证据，不复述全部数据。
3. 标注哪些影响因素被数据命中，哪些因素暂时不是主因。
4. 按优先级排序原因，不平铺所有可能性。
5. 给执行型 To-do，具体到门店、时段、商品、流程或人群。
6. 给验证标准：观察多久，什么结果算有效，什么结果需要换方案。

### 常用变量
- {问题现象}
- {核心指标变化}
- {命中因素}
- {优先级1/2/3}
- {执行任务}
- {验证指标}

### 合格检查点
- 是否先给初步判断/数据结论
- 是否解释原因在门店中如何发生
- 是否有原因优先级
- 是否有具体 To-do
- 是否有验证标准

## 判断决策型

### 无数据版回答结构
1. 先说明不能凭感觉或单一现象做决定。
2. 列出做/不做/先测试的判断条件。
3. 说明每种选择可能带来的风险和影响。
4. 告诉商户补充哪些信息后可以给更明确建议。
5. 给低风险测试方案，不直接建议大范围调整。

### 有数据版回答结构
1. 直接给建议：建议做、不建议做、先小范围测试，或换替代方案。
2. 用数据说明为什么做这个判断。
3. 说明风险点和边界条件。
4. 给测试方案：对象、范围、周期、指标。
5. 给继续/停止/扩大/缩小的判断标准。

### 常用变量
- {决策问题}
- {判断条件}
- {数据依据}
- {建议动作}
- {测试范围}
- {停止/继续标准}

### 合格检查点
- 是否给明确决策
- 是否说明什么情况该做/不该做
- 是否避免大范围盲调
- 是否给小范围测试
- 是否给验证指标

## 方案生成型

### 无数据版回答结构
1. 先给方案方向，不只讲原则。
2. 说明为什么这个方向适合当前问题。
3. 给方案结构，例如商品角色、套餐结构、活动目标、触达方式。
4. 引导商户提供信息，以便进一步细化。
5. 给执行步骤和时间节奏。

### 有数据版回答结构
1. 基于数据指出当前方案的主要问题。
2. 直接给优化方案，并明确调整对象。
3. 把动作写成任务：改哪个商品、哪个价位、哪个时段、哪个页面或哪个活动。
4. 给执行排期。
5. 给验证标准。

### 常用变量
- {方案目标}
- {当前问题}
- {方案结构}
- {调整对象}
- {执行步骤}
- {验证指标}

### 合格检查点
- 是否有明确方案结构
- 是否给执行步骤
- 是否能让商户今天开始做
- 是否避免只说优化
- 是否有复盘指标

## 方法教学型

### 无数据版回答结构
1. 先告诉商户不用复杂系统也能做初步判断。
2. 给一个简单可执行的方法或记录方式。
3. 说明每一步记录什么、怎么看。
4. 告诉商户做完后如何判断问题点。
5. 引导商户把记录结果发来做进一步分析。

### 有数据版回答结构
1. 基于已有记录判断当前处在哪个环节。
2. 指出记录中最明显的异常点。
3. 告诉商户下一步怎么优化。
4. 给数据对照方式和继续记录方法。
5. 给结果判断标准。

### 常用变量
- {观察/记录方法}
- {当前数据}
- {异常环节}
- {下一步优化}
- {继续记录指标}

### 合格检查点
- 是否给可操作方法
- 是否说明如何记录
- 是否说明如何判断
- 有数据时是否定位异常环节
- 是否给下一步优化

## 复盘评估型

### 无数据版回答结构
1. 先说明不能只看单一指标。
2. 列出评估维度，例如订单、毛利、新客、客单、复购、差评。
3. 告诉商户需要补充哪些前/中/后数据。
4. 给粗略判断方法。
5. 给下一次动作前必须记录的基准数据。

### 有数据版回答结构
1. 直接判断效果：好、一般、有风险、应停止/缩小/调整。
2. 用数据说明哪里有效、哪里无效。
3. 定位问题：利润、客单、新客、复购、接待能力或商品结构。
4. 给下次优化方案。
5. 给继续/停止/扩大/缩小的判断标准。

### 常用变量
- {复盘对象}
- {前/中/后数据}
- {效果判断}
- {有效点}
- {风险点}
- {下次调整}

### 合格检查点
- 是否不只看订单
- 是否有前后对比
- 是否判断有效/无效原因
- 是否给下次调整
- 是否给继续/停止标准

## 定位表达型

### 无数据版回答结构
1. 先说明不是所有优势都叫差异化。
2. 给差异化判断标准：顾客能感知、竞品不易复制、有证据支撑。
3. 让商户从顾客、竞品、评价中找证据。
4. 给表达方向。
5. 给落地位置：菜单、海报、社媒、员工话术、活动文案。

### 有数据版回答结构
1. 根据评价、竞品、顾客反馈判断最强差异点。
2. 排除不成立或用户未感知的差异点。
3. 给定位表达建议。
4. 给内容、菜单、活动和服务话术落地方案。
5. 给验证方式：用户是否开始复述该差异点。

### 常用变量
- {目标顾客}
- {竞品差异}
- {好评关键词}
- {差异点}
- {表达落地}
- {验证信号}

### 合格检查点
- 是否有目标顾客和竞品对比
- 是否有证据支撑差异
- 是否排除空泛卖点
- 是否给表达落地位置
- 是否给验证方式
