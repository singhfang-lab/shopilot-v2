# 商户助手知识库 V9：问题卡 + 商户端回答蓝图

本版本在 V8「问题卡 + 商户输出契约」基础上，新增 **C. 商户端回答蓝图**。

每个问题文件均包含三部分：

1. **A. 问题卡**：给 AI / 内容同事使用，固定分析逻辑、影响因素、数据策略、数据命中规则和禁止表达。
2. **B. 商户输出契约**：约束回答质量，规定无数据/有数据时必须包含什么、动作要写到什么颗粒度、如何验证。
3. **C. 商户端回答蓝图**：真正面向商户回答生成的参考结构，包含无数据版/有数据版回答结构、参考回答和合格检查点。

## 六类商户端回答蓝图

| 类型 | 适用问题 | 无数据时 | 有数据时 |
|---|---|---|---|
| 原因诊断型 | 为什么下降、为什么不复购、为什么某店不行 | 常见影响因素 + 门店发生方式 + 排查路径 | 数据结论 + 命中因素 + 原因优先级 + 执行任务 |
| 判断决策型 | 要不要降价、要不要下架、要不要继续推 | 判断条件 + 风险 + 低风险测试 | 明确建议 + 数据依据 + 测试方案 + 继续/停止标准 |
| 方案生成型 | 怎么设计菜单、套餐、活动、引流 | 方案方向 + 结构 + 执行步骤 | 基于数据指出问题 + 调整对象 + 执行排期 |
| 方法教学型 | 没数据怎么分析、怎么分析竞品、社媒怎么做 | 可操作方法 + 记录方式 + 判断规则 | 基于记录定位异常环节 + 下一步优化 |
| 复盘评估型 | 活动/新品/调价有没有效果 | 评估维度 + 需要补什么数据 | 效果判断 + 有效/无效原因 + 下次调整 |
| 定位表达型 | 差异化、目标顾客、定位、社媒表达 | 差异判断标准 + 用户/竞品/评价证据 | 最强差异点 + 表达落地 + 验证方式 |

## 使用方式

- 内容同事：优先看每个文件的 **C. 商户端回答蓝图**，理解最终商户应看到怎样的回答。
- 产品/AI同事：用 **A. 问题卡** 和 **B. 输出契约** 约束 AI 回答稳定性和质量。
- 测试同事：用 **C. 合格回答检查点** 判断 AI 输出是否足够具体。
